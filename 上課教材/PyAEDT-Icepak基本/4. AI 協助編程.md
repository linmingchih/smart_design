# 第四章 AI 協助編程與 AI Agent 應用

## 4.1 介紹 AI Agent 編程
在傳統的程式開發中，開發者需要手動撰寫每一行程式碼並自行處理所有邏輯。而在 AI 輔助開發的新時代中，**AI Agent（人工智慧代理）** 成為了提升生產力的關鍵。AI Agent 不僅僅是提供問答或程式碼補全，它具備自主思考、規劃與執行任務的能力，能夠根據使用者的指令，在特定環境中自動收集資訊、呼叫工具並完成複雜的編程任務。

透過 AI Agent 編程，開發者可以將重心從「如何寫出這段程式碼」轉移到「如何清晰地定義目標與流程」，讓 AI 負責繁瑣的實作細節。

## 4.2 何謂 Skill？
在 AI Agent 的架構中，**Skill（技能）** 是指賦予 Agent 特定領域知識或操作能力的模組化工具。如同人類學習新技能來解決特定問題一樣，我們可以安裝不同的 Skills 讓 AI Agent 看懂特定格式的檔案、理解特定的 API 語法或是操控某些專屬軟體（例如 PyAEDT 或 PyEDB）。

透過引入專業領域的 Skill，AI Agent 在回答問題或生成程式碼時，就能夠更精準地符合該領域的規範與最佳實踐。

## 4.3 新增 Skill 教學
為了讓 AI Agent 能夠更好地協助我們進行 PyEDB 相關的自動化設計，我們可以安裝社群或專家所開發好的 Skill。

請在您的終端機（Terminal）或命令提示字元中，執行以下指令來新增 `pyedb-skill`：

```bash
npx skills add https://github.com/linmingchih/pyedb-skill
```

這項指令會透過 `npx` 工具，從指定的 GitHub 儲存庫下載並安裝 PyEDB 的專屬技能。安裝完成後，您的 AI Agent 就會具備處理 PyEDB 相關任務的先備知識，在後續的腳本編寫與自動化流程中，提供更準確的程式碼建議與技術支援。

## 4.4 AI Engineering 核心概念 (AI Engineering, Prompt, Context, Harness)
在應用 AI Agent 協助編程的過程中，我們實際上是在進行 **AI Engineering (AI 工程)**。AI 工程是一門將人工智慧模型（如 LLM）整合到實際軟體系統與工作流程中的學科，其目標是確保 AI 的輸出穩定、可靠且具備商業價值。在這個過程中，有三個關鍵要素：

### 1. Prompt (提示詞)
**Prompt** 是我們給予 AI 的指令或問題，是引導 AI 產生預期結果的起點。
在 AI 工程中，Prompt 的設計不僅僅是「問問題」，更是一門藝術（Prompt Engineering）。一個好的 Prompt 應該清晰、具體，並包含充分的約束條件（例如：「請用 Python 撰寫一個迴圈，並且加上註解，不要使用外部套件」）。這能有效降低 AI 產生幻覺（Hallucination）或給出偏離主題答案的機率。

### 2. Context (上下文/語境)
**Context** 是 AI 在生成回答時所參考的背景資訊。
由於語言模型本身通常不會持久保存對話外的狀態，我們必須將所有相關的資訊（如先前的對話紀錄、專案架構、特定的程式碼片段或錯誤訊息）作為 Context 一起提供給它。在 AI Agent 的架構中，Agent 會自動幫我們收集並管理 Context（例如自動讀取工作目錄下的檔案、查詢相關資料），讓 AI 能夠根據當下最真實的「語境」給出精準的建議，而不是憑空捏造。

### 3. Harness (測試與評估框架)
**Harness** (或稱 Test Harness / Evaluation Harness) 是用來測試、評估與監控 AI 系統表現的框架與腳本環境。
在傳統軟體開發中有單元測試，在 AI 工程中同樣需要 Harness 來確保 AI 的輸出符合預期。透過 Harness，我們可以建立一套標準的評估指標與自動化測試案例。當我們修改了 Prompt、替換了背後的語言模型，或是更新了 Context 的檢索方式時，Harness 能夠幫助我們量化這些改變是否帶來了正向的效果，從而確保 AI 系統的穩定性與持續迭代的能力。
