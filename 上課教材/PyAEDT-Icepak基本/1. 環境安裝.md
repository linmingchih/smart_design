# 第1章 環境安裝與 Python 生態系準備

在開始自動化熱流分析之前，我們必須先準備好執行環境。現代的工程自動化高度依賴開源生態系，因此本章節將帶您初步了解 Python 的背後運作機制，學習如何建立乾淨的虛擬環境，並透過現代化的套件管理工具安裝控制 AEDT Icepak 所需的核心模組。

---

## 1.1 Python 生態系簡介

在撰寫第一行程式碼之前，我們需要認識幾個構成 Python 強大開源生態系的關鍵名詞：

*   **Python**：一種高階、直譯式且強調程式碼可讀性的程式語言。由於其易學易用的特性，已成為資料科學、人工智慧與工程自動化領域的標準語言。
*   **PSF (Python Software Foundation)**：Python 軟體基金會。這是一個非營利組織，負責推廣、保護和推動 Python 程式語言的發展，確保其作為開源軟體的地位與社群的活躍。
*   **PyPI (Python Package Index)**：Python 官方的第三方軟體儲存庫。您可以將它想像成 Python 的「App Store」，全世界的開發者都可以將自己寫好的套件上傳至此，讓其他人免費下載使用。
*   **PyANSYS**：這是 ANSYS 官方推動的開源專案。ANSYS 將旗下許多工程軟體（包含 AEDT, Mechanical, Fluent 等）的 API 封裝成 Python 套件並發布在 PyPI 上。透過 PyANSYS，工程師能以現代化的 Python 語法來控制傳統的 CAE 軟體，大幅降低自動化與跨領域整合的門檻。我們即將使用的 `ansys-aedt-core` (即 PyAEDT) 便是 PyANSYS 生態系中的重要一員。

---

## 1.2 虛擬環境 (Virtual Environment) 的重要性

當您在進行多個專案時，專案 A 可能需要使用 `ansys-aedt-core` 版本 0.6，而專案 B 卻需要版本 0.7。如果把所有套件都安裝在系統全域，很容易產生「版本衝突」的問題。

**虛擬環境** 的概念就是在您的電腦中劃分出一個「獨立且隔離的空間」。在這個空間裡，擁有專屬的 Python 執行檔與套件庫。為每一個專案建立各自的虛擬環境，是 Python 開發中最核心的良好習慣。

---

## 1.3 套件管理工具：pip 與 uv

要在虛擬環境中下載並安裝 PyPI 上的套件，我們需要「套件管理工具」。

*   **pip**：Python 內建且最標準的套件管理工具。幾乎所有的 Python 基礎教學都會教您使用 `pip install <package_name>` 來安裝套件。
*   **uv**：近年來由 Astral 開發，以 Rust 語言撰寫的極速 Python 封裝與環境管理工具。它的操作指令與 `pip` 幾乎完全相容，但在解析相依性與下載安裝套件的速度上，往往能比原生 `pip` 快上數十倍。在企業級的大型專案或 CI/CD 流程中，越來越多團隊轉向使用 `uv`。

> **💡 如何在系統中安裝 uv？**  
> 如果您的電腦尚未安裝 `uv`，可以透過以下官方指令快速安裝 (只需執行一次)：
> *   **Windows (PowerShell)**: 
>     ```powershell
>     powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
>     ```
> *   **macOS / Linux (Terminal)**: 
>     ```bash
>     curl -LsSf https://astral.sh/uv/install.sh | sh
>     ```

## 1.4 建立環境與安裝 PyAEDT

接下來，我們將實際建立一個虛擬環境，並安裝自動化 Icepak 所需的核心套件。您可以選擇使用傳統標準的 `pip` 流程，或是體驗極速的 `uv`。

### 選項 A：使用標準的 Python 與 pip

1. 開啟命令提示字元或終端機，並導航至您的專案資料夾。
2. 建立名為 `.venv` 的虛擬環境：
   ```bash
   python -m venv .venv
   ```
3. 啟動虛擬環境 (Windows 系統)：
   ```bash
   .venv\Scripts\activate
   ```
4. 安裝 PyANSYS 核心套件與後處理所需庫：
   ```bash
   pip install ansys-aedt-core pandas pyvista
   ```

### 選項 B：使用極速的 uv (推薦)

如果您已經在系統上安裝了 `uv`，我們強烈推薦使用其現代化的專案管理流程。它會自動為您處理 `pyproject.toml` 配置檔並建立 `.venv` 虛擬環境：

1. 開啟命令提示字元或終端機，並導航至您的專案空資料夾。
2. 初始化專案並指定 Python 版本（例如 3.10）：
   ```bash
   uv init --python 3.10
   ```
   *(這會建立基本的專案結構，包含 `pyproject.toml`)*
3. 加入並安裝 PyAEDT (包含所有擴充套件) 以及後處理套件：
   ```bash
   uv add pyaedt[all] pandas pyvista
   ```
   *(`uv` 會自動建立 `.venv` 虛擬環境，並極速解析、下載、安裝所有套件並更新 `pyproject.toml`)*
4. (選用) 若您是從版本控制 (如 Git) 下載已包含 `pyproject.toml` 的專案，只需執行同步指令即可自動還原環境：
   ```bash
   uv sync
   ```
5. 啟動虛擬環境 (Windows 系統)：
   ```bash
   .venv\Scripts\activate
   ```

*(註：`pandas` 與 `pyvista` 將在後續章節的後處理及圖表繪製時發揮極大作用，強烈建議一併安裝。)*
