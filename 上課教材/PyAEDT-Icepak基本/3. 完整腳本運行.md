# 第3章 完整腳本運行：顯示卡熱分析

本章節將帶領您透過一個完整的 Python 腳本範例，了解如何使用 PyAEDT 在 Icepak 中執行顯示卡（Graphic Card）的熱分析（Thermal Analysis）。

這個範例涵蓋了從環境初始化、模型下載、邊界條件設定、網格劃分、監測點設定、求解器設定，到最終的資料後處理（包含定量與定性分析）的完整流程。透過此範例，您將能熟悉 PyAEDT Icepak 的核心自動化功能。

---

## 3.1 匯入模組與定義常數

**📝 程式碼概要解析：**
在自動化腳本的起點，我們首先需要引入負責處理檔案系統與時間的標準函式庫 (`os`, `tempfile`, `time`)、負責數值與資料處理的套件 (`pandas`)，以及核心的 `ansys.aedt.core`。
同時，我們會宣告一些全域常數，例如 AEDT 版本號、運算核心數，以及是否開啟圖形介面 (`NG_MODE`)。將介面關閉 (Non-Graphical Mode) 可以大幅提升腳本在背景執行的速度並節省系統資源。

```python
import os
import tempfile
import time

import ansys.aedt.core
import pandas as pd
from ansys.aedt.core.examples.downloads import download_icepak
from IPython.display import Image

# 定義常數
AEDT_VERSION = "2026.1"  # 設定您當前使用的 AEDT 版本
NUM_CORES = 4            # 設定模擬時使用的核心數
NG_MODE = False          # 是否以無圖形介面 (Non-Graphical) 模式執行
```

> **提示**：將 `NG_MODE` 設為 `True` 可以在背景執行 AEDT，這在伺服器端或進行大量批次模擬時非常實用。

---

## 3.2 建立工作目錄與載入專案

**📝 程式碼概要解析：**
為確保測試環境乾淨且不污染既有的工作目錄，程式會呼叫 `tempfile` 自動建立一個暫存資料夾。接著，利用內建的 `download_icepak` 方法從網路上下載預設的顯示卡散熱範例專案。最後，將專案路徑、版本資訊等參數傳入 `ansys.aedt.core.Icepak()` 類別，正式啟動 AEDT Icepak 實例。

```python
# 建立暫存資料夾以儲存下載與輸出的資料
temp_folder = tempfile.TemporaryDirectory(suffix=".ansys")

# 下載預先建立好幾何模型與材質的 Icepak 專案
project_temp_name = download_icepak(local_path=temp_folder.name)

# 啟動 Icepak 專案
ipk = ansys.aedt.core.Icepak(
    project=project_temp_name,
    version=AEDT_VERSION,
    new_desktop=True,
    non_graphical=NG_MODE,
)
```

## 3.3 模型視覺化與旋轉

**📝 程式碼概要解析：**
PyAEDT 內建了強大的 PyVista 渲染引擎。這段程式碼展示了如何在不依賴 AEDT 原生圖形介面的情況下，直接透過 Python 匯出 3D 模型的截圖。
我們首先會呼叫 `ipk.plot()` 輸出當前視角的幾何圖片，並且特別把 `plot_air_objects` 設為 `False` 讓空氣域隱藏以看清內部；接著利用 `rotate` 方法將模型繞著 X 軸旋轉，最後再輸出第二張旋轉後的圖片，藉此快速確認模型匯入的狀態。

```python
# 輸出初始模型視角
plot1 = ipk.plot(
    show=False,
    output_file=os.path.join(temp_folder.name, "Graphics_card_1.jpg"),
    plot_air_objects=False,  # 隱藏空氣域以便看清楚內部結構
)

# 繞著 X 軸旋轉所有物件
ipk.modeler.rotate(ipk.modeler.object_names, "X")

# 輸出旋轉後的模型視角
plot2 = ipk.plot(
    show=False,
    output_file=os.path.join(temp_folder.name, "Graphics_card_2.jpg"),
    plot_air_objects=False,
)
```

## 3.4 邊界條件設定 (Boundary Conditions)

### 3.4.1 設定發熱源

**📝 程式碼概要解析：**
熱源是熱傳分析的核心驅動力。我們利用 `assign_solid_block` 方法，精準選取 3D 幾何物件（CPU 和兩個記憶體晶片），並賦予它們固定的發熱功率 (`power_assignment`)，藉此模擬這些晶片在實際運作時產生的熱能。

```python
# 將 CPU 設定為 25W 的固態發熱區塊
ipk.assign_solid_block(object_name="CPU", power_assignment="25W")

# 將兩個記憶體物件設定為 5W 的固態發熱區塊
ipk.assign_solid_block(object_name=["MEMORY1", "MEMORY1_1"], power_assignment="5W")
```

### 3.4.2 設定進出口條件

**📝 程式碼概要解析：**
為了模擬電腦機殼或風洞內的空氣流動，我們需要定義流體域 (Region) 的進出口條件。透過提取 `Region` 物件，我們指定其正 X 面 (`top_face_x`) 為自由開放的壓力出口 (Pressure Outlet)；同時將負 X 面 (`bottom_face_x`) 設定為給定風速的流速入口 (Velocity Inlet)，這代表著從該方向吹入 1 m/s 的冷卻氣流。

```python
region = ipk.modeler["Region"]

# 設定 Region 的正 X 面 (top_face_x) 為自由開口的壓力出口
ipk.assign_pressure_free_opening(
    assignment=region.top_face_x.id, 
    boundary_name="Outlet"
)

# 設定 Region 的負 X 面 (bottom_face_x) 為進風口，流速為 X 方向 1 m/s
ipk.assign_velocity_free_opening(
    assignment=region.bottom_face_x.id,
    boundary_name="Inlet",
    velocity=["1m_per_sec", "0m_per_sec", "0m_per_sec"],
)
```

## 3.5 網格設定 (Mesh Settings)

**📝 程式碼概要解析：**
網格品質直接影響熱流分析的精確度。針對溫度梯度與流場變化劇烈的關鍵元件（如散熱器與 CPU），我們會建立局部網格控制 (`Mesh Region`)。
程式碼中將網格設定切換為手動 (`manual_settings = True`)，不僅啟用了多層次網格 (`EnableMLM`) 來優化網格過渡，還限制了 XYZ 方向的最大網格尺寸 (2mm)。最後，利用 `Padding` 參數設定了網格區域相對於物件本體的外擴邊距，確保關鍵熱源周圍有一層足夠密集的網格解析度。

```python
# 指定散熱器與 CPU 建立局部網格區域
mesh_region = ipk.mesh.assign_mesh_region(assignment=["HEAT_SINK", "CPU"])

# 將網格設定改為手動控制
mesh_region.manual_settings = True

# 修改各軸向的最大網格尺寸及多層次網格 (MLM) 參數
mesh_region.settings["MaxElementSizeX"] = "2mm"
mesh_region.settings["MaxElementSizeY"] = "2mm"
mesh_region.settings["MaxElementSizeZ"] = "2mm"
mesh_region.settings["EnableMLM"] = True
mesh_region.settings["MaxLevels"] = "2"
mesh_region.settings["MinElementsInGap"] = 4
mesh_region.update()

# 調整網格區域相對於物件周圍的外擴邊距 (Padding)
subregion = mesh_region.assignment
subregion.positive_x_padding = "20mm"
subregion.positive_y_padding = "5mm"
subregion.positive_z_padding = "5mm"
subregion.negative_x_padding = "5mm"
subregion.negative_y_padding = "5mm"
subregion.negative_z_padding = "10mm"
```

## 3.6 監測點設定 (Monitors)

**📝 程式碼概要解析：**
監測點讓我們能在運算過程中即時觀察收斂狀況，或在完成後快速萃取特定位置的數值。這裡我們建立了兩種監測點：
1. **面監測點 (Face Monitor)**：針對 CPU 上方面 (`top_face_z`)，也就是與散熱器的接觸面，監測其平均溫度。
2. **點監測點 (Point Monitor)**：利用 `for` 迴圈，在散熱器下游沿著 X 軸每隔 2mm 佈建一個點，用以觀察風速隨距離的空間變化。

```python
# 設定面監測點：監測 CPU 與散熱器接觸面 (top_face_z) 的溫度
cpu = ipk.modeler["CPU"]
m1 = ipk.monitor.assign_face_monitor(
    face_id=cpu.top_face_z.id,
    monitor_quantity="Temperature",
    monitor_name="TemperatureMonitor1",
)

# 設定點監測點：在下游沿著 X 軸每隔 2mm 設置一個流速監測點
speed_monitors = []
for x_pos in range(0, 10, 2):
    m = ipk.monitor.assign_point_monitor(
        point_position=[f"{x_pos}mm", "40mm", "15mm"], 
        monitor_quantity="Speed"
    )
    speed_monitors.append(m)
```

## 3.7 專案求解 (Solve Project)

**📝 程式碼概要解析：**
物理條件設定完畢後，必須建立分析設定檔 (Setup)。這段程式碼將流場定義為紊流 (`Turbulent`) 以符合高速冷卻氣流的物理特性。為了教材演示能快速跑完，我們將最大迭代次數限制為 5 次。此外，將壓力和溫度的線性求解器切換為彈性模式 (`flex`)。設定完成後儲存專案，並指派平行運算的核心數正式啟動求解。

```python
# 建立預設的求解設定
setup1 = ipk.create_setup()

# 調整流態為紊流 (Turbulent)，最大迭代次數設為 5 次以便快速演示
setup1.props["Flow Regime"] = "Turbulent"
setup1.props["Convergence Criteria - Max Iterations"] = 5

# 設定壓力與溫度的線性求解器類型
setup1.props["Linear Solver Type - Pressure"] = "flex"
setup1.props["Linear Solver Type - Temperature"] = "flex"

# 儲存並開始分析
ipk.save_project()
ipk.analyze(setup=setup1.name, cores=NUM_CORES, tasks=NUM_CORES)
```

## 3.8 後處理分析 (Postprocessing)

### 3.8.1 定量分析 (Quantitative Analysis)

**📝 程式碼概要解析：**
模擬完成後，傳統流程需要手動匯出報表，但透過 PyAEDT 我們能將數據直接拉進 Python 資料科學神器——Pandas DataFrame 之中。
這段冗長的分析腳本依序執行了：
1. 讀取 CPU 面監測點的溫度極值與平均值。
2. 透過 `Field Summary` 批次擷取那排空間點的流速與溫度，並將空間座標 (X, Y, Z) 補回表格中。
3. 利用 Pandas 內建繪圖功能畫出「流速 vs. X 軸位置」的折線圖。
4. 使用 `pd.merge()` 將流速與溫度兩份資料表合併，並計算它們之間的相關係數 (Correlation)，最後繪製散佈圖來確認「流速越快是否對應溫度越低」的物理現象。

```python
# 獲取 CPU 面監測點的溫度數據 (回傳包含 'Min', 'Max', 'Mean' 的字典)
temperature_data = ipk.post.evaluate_monitor_quantity(monitor=m1, quantity="Temperature")
print(temperature_data)

# 透過 Field Summary 擷取多個速度監測點的資料並轉換為 Pandas DataFrame
speed_fs = ipk.post.create_field_summary()
for m_name in speed_monitors:
    speed_fs.add_calculation(entity="Monitor", geometry="Volume", geometry_name=m_name, quantity="Speed")
speed_data = speed_fs.get_field_summary_data(pandas_output=True)

# 補上監測點的空間座標 (X, Y, Z) 以便進行作圖
for i in range(3):
    direction = ["X", "Y", "Z"][i]
    speed_data["Position" + direction] = [ipk.monitor.all_monitors[entity].location[i] for entity in speed_data["Entity"]]

# 使用 Pandas 繪製不同 X 座標下的流速變化曲線
speed_data.plot(
    x="PositionX",
    y="Mean",
    kind="line",
    marker="o",
    ylabel=speed_data.at[0, "Quantity"],
    xlabel=f"x [{ipk.modeler.model_units}]",
    grid=True,
)

# 接著，以同樣的方式擷取這些位置的溫度數據
temperature_fs = ipk.post.create_field_summary()
for m_name in speed_monitors:
    temperature_fs.add_calculation(entity="Monitor", geometry="Volume", geometry_name=m_name, quantity="Temperature")
temperature_fs = temperature_fs.get_field_summary_data(pandas_output=True)

# 使用 pd.merge 將溫度與速度的 DataFrame 合併，便於比較
merged_df = pd.merge(temperature_fs, speed_data, on="Entity", suffixes=("_temperature", "_speed"))

# 整理欄位名稱並計算流速與溫度之間的相關係數 (Correlation)
temperature_quantity = temperature_fs["Quantity"].iloc[0]
velocity_quantity = speed_data["Quantity"].iloc[0]
merged_df.rename(
    columns={"Mean_temperature": temperature_quantity, "Mean_speed": velocity_quantity},
    inplace=True,
)

correlation = merged_df[velocity_quantity].corr(merged_df[temperature_quantity])

# 繪製流速與溫度的散佈圖
ax = merged_df.plot.scatter(x=velocity_quantity, y=temperature_quantity)
ax.set_title(f"Correlation between Temperature and Velocity: {correlation:.2f}")
```

從分析中可以觀察到，遠離發熱組件時，由於空氣混合與流動，空氣流速較快且溫度較低。這個例子充分展示了將場資料導入 Pandas 進行資料科學分析的潛力。

### 3.8.2 定性分析 (Qualitative Analysis)

**📝 程式碼概要解析：**
直觀的視覺雲圖是工程溝通不可或缺的環節。這段程式碼展示兩種出圖方式：
1. **表面溫度雲圖**：蒐集 CPU 與記憶體所有表面 ID，建立溫度雲圖 (`SurfTemperature`)，設定好視角後直接輸出成 PNG 圖片。
2. **3D 空間溫度場**：選擇多個關鍵零組件，呼叫 PyVista (`plot_field`) 繪製出帶有 CAD 幾何特徵的高品質 3D 溫度場分布，並將其結果匯出到暫存資料夾中。

```python
# 獲取 CPU 與記憶體的所有表面 ID
surflist = [i.id for i in ipk.modeler["CPU"].faces]
surflist += [i.id for i in ipk.modeler["MEMORY1"].faces]
surflist += [i.id for i in ipk.modeler["MEMORY1_1"].faces]

# 建立表面溫度雲圖 (SurfTemperature)
plot3 = ipk.post.create_fieldplot_surface(assignment=surflist, quantity="SurfTemperature")

# 匯出結果為 PNG 圖片並顯示
path = plot3.export_image(
    full_path=os.path.join(temp_folder.name, "temperature.png"),
    orientation="top",
    show_region=False,
)
Image(filename=path)

# 使用 PyVista 在外部顯示這些物件的 3D 溫度場
plot4 = ipk.post.plot_field(
    quantity="Temperature",
    assignment=[
        "SERIAL_PORT", "MEMORY1", "MEMORY1_1", "CAPACITOR", 
        "CAPACITOR_1", "KB", "HEAT_SINK", "CPU", "ALPHA_MAIN_PCB"
    ],
    plot_cad_objs=False,
    show=False,
    export_path=temp_folder.name,
)
```

## 3.9 釋放資源與清理

**📝 程式碼概要解析：**
自動化腳本最容易被忽略卻最重要的一步，就是釋放授權與資源。分析與出圖結束後，程式會先儲存專案，接著呼叫 `release_desktop()` 關閉 AEDT 的背景程序。為避免檔案被系統佔用，我們讓腳本暫停 3 秒鐘，最後才呼叫 `cleanup()` 刪除一開始建立的暫存資料夾，確保不留系統垃圾。

```python
# 儲存專案並關閉 AEDT Desktop
ipk.save_project()
ipk.release_desktop()

# 等待 3 秒鐘，確保 AEDT 已完全關閉
time.sleep(3)

# 刪除暫存目錄及其中的所有檔案
temp_folder.cleanup()
```
