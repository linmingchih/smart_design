## 2.1 啟動並連接 AEDT Icepak

PyAEDT 提供了一個高度封裝的 `Icepak` 類別，讓我們能輕鬆啟動軟體或連接到已開啟的專案。

**📝 程式碼概要解析：**
要啟動 Icepak 實例，我們需要從 `ansys.aedt.core` 匯入 `Icepak` 類別。在實例化時，您可以傳入多個關鍵參數：
*   `version`：指定要啟動的 AEDT 版本（例如 `"2023.2"` 或 `"2024.1"`）。
*   `non_graphical`：設為 `False` 會顯示圖形介面，適合除錯與學習；設為 `True` 則在背景執行，適合批次運算。
*   `new_desktop_session`：設為 `True` 代表強制開啟一個全新的 AEDT 視窗。

```python
from ansys.aedt.core import Icepak

# 初始化 Icepak 實例
ipk = Icepak(
    version="2024.1", 
    non_graphical=False, 
    new_desktop_session=True,
    projectname="My_Thermal_Project",  # 可選：建立新專案名稱
    designname="Icepak_Design_1"       # 可選：建立新設計名稱
)

print(f"成功連接 AEDT! 目前專案名稱為: {ipk.project_name}")
```

## 2.2 探索 API：type()、dir() 與 help()

PyAEDT 是一個功能極為龐大的函式庫，要記住所有的指令是不切實際的。學習如何使用 Python 的內建函數來「探索」未知的物件與方法，是撰寫自動化腳本及 AI 輔助編程的核心技能。

**📝 程式碼概要解析：**
*   **`type()`**：用來確認目前操作的變數是屬於什麼「類別 (Class)」。這對於搞清楚「我現在到底拿到什麼物件」非常重要。
*   **`dir()`**：會條列出該物件底下「所有可用的屬性與方法」。當你忘記某個功能的確切拼法時，可以用它來快速搜索。
*   **`help()`**：讀取並印出該方法的官方說明文件 (Docstring)，這會詳細告訴您這個函數需要什麼參數、預設值是什麼，以及會回傳什麼。

```python
# 1. 查詢物件型別
print("ipk 的型別是:", type(ipk))

# 2. 列出 ipk 物件下的所有方法與屬性 (這裡以擷取前 10 個為例)
all_attributes = dir(ipk)
print("\nipk 的前 10 個屬性/方法:", all_attributes[:10])

# 實用技巧：結合 list comprehension 尋找特定的指令
# 例如：找尋所有名稱中包含 'assign' 的方法
assign_methods = [method for method in dir(ipk) if 'assign' in method]
print("\nipk 中與 assign 相關的方法:", assign_methods[:5])

# 3. 查看特定方法的詳細說明文件 (會顯示參數格式與用法)
help(ipk.assign_solid_block)
```

## 2.3 PyAEDT 的物件導向結構 (OOP)

PyAEDT 採用了非常直觀的階層式**物件導向 (Object-Oriented Programming)** 設計。它將原本散落在 AEDT 圖形介面各個選單中的功能，分門別類地封裝成不同的子模組 (Sub-modules)。

主要的子模組包含：
*   **`ipk.modeler`**：負責 3D 幾何建模、布林運算、物件選擇與平移旋轉。
*   **`ipk.mesh`**：負責全域網格設定以及局部網格 (Mesh Region) 劃分。
*   **`ipk.materials`**：管理材料庫、新增材料、指派材料給幾何物件。
*   **`ipk.monitor`**：負責建立溫度、速度等點/線/面監測點。
*   **`ipk.post`**：專司後處理，負責萃取數據、繪製圖表與 3D 雲圖。

**📝 程式碼概要解析：**
透過這種「點 (`.`)」來呼叫下層模組的語法，您可以很輕易地猜出某個功能藏在哪裡。例如建立一個幾何方塊，直覺上它屬於「建模」範疇，因此會放在 `ipk.modeler` 底下。

```python
# 透過 modeler 子模組建立一個 3D 方塊
box = ipk.modeler.create_box(
    origin=[0, 0, 0], 
    sizes=[10, 20, 5], 
    name="CPU_Die", 
    material="silicon"
)

# 透過 materials 子模組修改材料的熱傳導係數
# (註：如果是內建材料通常不可改，需先複製或建立自訂材料)
my_mat = ipk.materials.add_material("Custom_Silicon")
my_mat.thermal_conductivity = 148

# 透過 post 子模組進行後處理資料擷取
# ipk.post.evaluate_monitor_quantity(...)
```

## 2.4 參數的基本設置方式

在 AEDT Icepak 的熱分析中，輸入的值通常帶有物理單位（例如 `W`, `mm`, `m/s`）。PyAEDT 設計了非常優雅的機制來處理這些帶單位的參數。

### 2.4.1 使用帶單位的字串

**📝 程式碼概要解析：**
在大多數的 PyAEDT 函數中，如果您只輸入數字（例如 `10`），系統會預設使用專案當前的單位；為了確保腳本不受環境預設值的影響，**強烈建議直接輸入帶單位的「字串」**。

```python
# 建立風扇邊界條件，明確指定字串單位
ipk.assign_fan(
    assignment="Fan_Face",
    boundary_name="Cooling_Fan",
    is_2d=True,
    speed="3000rpm"      # 明確指定為 rpm
)

# 設定發熱區塊，指定為 15 瓦特
ipk.assign_solid_block(object_name="CPU_Die", power_assignment="15W")
```

### 2.4.2 專案變數 (Variables) 設定

**📝 程式碼概要解析：**
參數化建模是自動化的核心價值。在 PyAEDT 中，設定變數與 Python 操作字典 (Dictionary) 的語法一模一樣。
以 `$` 開頭的變數代表「全域變數 (Project Variable)」；而沒有 `$` 開頭的則是當前設計專用的「局部變數 (Design Variable)」。

```python
# 設定設計變數 (Design Variable)
ipk["Chip_Power"] = "25W"
ipk["HeatSink_Width"] = "50mm"

# 設定全域變數 (Project Variable，所有 Design 皆可共用)
ipk["$Ambient_Temp"] = "25cel"

# 這些變數隨後可直接套用於邊界條件或幾何尺寸中
ipk.assign_solid_block(object_name="CPU_Die", power_assignment="Chip_Power")

# 如果想將 AEDT 中的變數值萃取回 Python 成為純數字
# 可以利用 evaluate_variable_value 函數
val_in_meters = ipk.evaluate_variable_value(variable_name="HeatSink_Width")
print(f"HeatSink_Width 在後台中轉換為純數字的值為: {val_in_meters}")
```
