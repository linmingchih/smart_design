# 利用 AI 探索陌生模組

面對一個全然陌生的 Python 模組（例如前一章節提到的 `ansys-mechanical-core`），直接從頭閱讀龐大的官方文件往往讓人不知所措，甚至容易放棄。最有效率的學習與開發方式是：**從官方範例開始，讓 AI 成為你的專屬原始碼分析師。**

## 核心策略：先見林，再見樹

不要試圖一次讀懂幾百行的程式碼。你可以將範例程式碼（或是範例的網址）直接提供給 AI，讓 AI 先幫你梳理整體的流程架構，接著再透過「針對性提問」，讓 AI 吐出特定功能的程式碼片段供你測試與執行。

### 實戰步驟（以 PyMechanical 閥門範例為例）

#### 1. 讓 AI 分析整體範例架構
首先，將整個範例丟給 AI。
* **Prompt 示範：** 「我現在要學習 `ansys-mechanical-core`，這裡有一個閥門靜力分析的範例 [貼上程式碼或網址]。請幫我總結這個範例的主要流程，列出它依序做了哪些重要設定。」

#### 2. 透過提問，提取並學習關鍵程式碼
在對整體流程有了基本概念後，針對你關心的物理設定或操作細節向 AI 提問。讓 AI 幫你從茫茫程式碼中「萃取」出可以獨立執行的片段：

* **如何加入材料 (Materials)？**
  * **Prompt 示範：** 「在這個範例中，如何從資料庫中加入『結構鋼 (Structural Steel)』？又是如何將這個材料指派 (assign) 給特定的閥門幾何實體？請列出相關的 Python 程式碼並逐行解釋。」

* **如何設定激勵或負載 (Excitations / Loads)？**
  * **Prompt 示範：** 「請抽出這個範例中施加負載的程式碼。它是如何在閥門內部施加壓力 (Pressure) 的？數值大小和作用的面是如何定義的？」

* **如何設立邊界條件 (Boundary Conditions)？**
  * **Prompt 示範：** 「針對邊界條件，這段範例是如何設定『固定支撐 (Fixed Support)』與『無摩擦支撐 (Frictionless Support)』的？請寫出對應的程式碼，並說明如何透過 API 選取到正確的幾何面。」

* **如何輸出與擷取結果 (Results)？**
  * **Prompt 示範：** 「求解完成後，如何輸出結果？請列出如何提取『總變形量 (Total Deformation)』與『等效應力 (Equivalent Stress)』的程式碼，並示範如何將結果匯出或顯示為圖片。」

#### 3. 逐步執行與拼湊
拿到 AI 提取的程式碼片段後，將它們放入你的編輯器（如 VS Code）或 Jupyter Notebook 中。
* **逐步執行：** 不要一次全跑，一段一段執行（例如：先確定材料有成功加上去，再試著加邊界條件），並觀察軟體內的變化或終端機的輸出。
* **修改參數：** 試著把 AI 給的壓力數值放大，或是更換材料名稱，看看結果有何不同。透過這種「提問 -> 執行 -> 修改」的迴圈，你能以最快的速度掌握一個陌生模組的核心用法。

## 結論：成功執行的意義

如果你能夠順利將這些拆解出來的程式碼片段跑完，並且得到預期的結果，這代表了三個重要的里程碑：

1. **環境與連線驗證：** 你的 Python 開發環境與底層模擬軟體（如 Ansys Mechanical）的連結是成功的，API 能夠正確驅動軟體。
2. **建立知識庫（Context）：** 這些成功執行的 API 呼叫方法與程式碼結構，將成為你未來要求 AI 建立其他更複雜範例時的絕佳「參考語境 (Context)」。AI 將能更準確地生成符合該模組風格的程式碼。
3. **固化成 Skill (技能)：** 最終，你可以將這些經過驗證的關鍵語法與流程歸納整理，建立成專屬的 `skill.md`（或類似的知識文件）。未來在面對類似需求時，就能直接調用這份文件，大幅提升開發效率與 AI 的生成品質。
