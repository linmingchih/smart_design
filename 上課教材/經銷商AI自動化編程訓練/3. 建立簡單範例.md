# 建立簡單範例並讓 AI 自行除錯

在熟悉了基礎的模組操作與 API 呼叫後，下一步是運用 AI 的生成能力，讓它為你建立一個全新的、從頭到尾完整的自動化專案。為了確保你能夠掌握全局，並驗證 AI 產生的程式碼品質，我們應該從一個**你本身非常熟悉且專長的領域**開始著手。

## 為什麼要從「自己專長」的範例開始？

* **容易判斷對錯：** 既然是你熟悉的領域，當 AI 生成物理設定、參數或流程時，你一眼就能看出邏輯是否正確、設定是否合理。
* **降低學習雙重障礙：** 如果讓 AI 寫一個你不懂的物理模擬，當程式報錯或結果異常時，你會分不清是「程式寫錯」還是「物理觀念錯誤」。
* **建立對 AI 的信任度：** 透過逐步引導 AI 完成你熟悉的專案，你可以清楚掌握 AI 的能力邊界，並建立一套專屬的協作模式。

## 實戰流程：讓 AI 自己寫、自己跑、自己除錯

這個階段的核心概念是將 AI 視為一個「初階工程師」，你則是「架構師」與「Code Reviewer」。你提供需求，AI 寫 Code，如果報錯，把錯誤訊息丟給 AI，讓它自己修正。

### 步驟 1：定義一個簡單且明確的專案需求

選擇一個你平時經常手動操作，且步驟不超過 5-10 步的簡單流程。

**（以 Python 與 PyMechanical 為例的 Prompt 示範）：**
> 「我需要使用 `ansys-mechanical-core` (PyMechanical) 撰寫一個自動化腳本。請幫我建立一個完整的 Python 專案，目標是進行一個簡單的『懸臂樑 (Cantilever Beam)』靜力分析。
>
> 具體需求如下：
> 1. 建立一個長度 100mm、截面為 10x10mm 的長方體幾何。
> 2. 將材料設定為 'Structural Steel'。
> 3. 對幾何體進行基礎網格劃分。
> 4. 在樑的一端（面）設定 Fixed Support。
> 5. 在樑的另一端施加 500 N 的向下力量 (Force)。
> 6. 求解並輸出最大總變形量 (Total Deformation) 的數值，並將結果雲圖存成圖片。
>
> 請提供完整的 Python 程式碼，確保它可以直接執行。」

### 步驟 2：審查程式碼邏輯 (Code Review)

收到 AI 給的程式碼後，**先不要急著執行**，先用你專業的眼光掃描一次：
* 幾何建立的 API 對嗎？
* 固定端和受力端的面選取邏輯是否合理？（這通常是 AI 最容易出錯的地方）
* 單位是否一致？

如果你發現明顯的邏輯錯誤，直接糾正它：
> 「你這段選取受力面的程式碼寫錯了，這樣會選到所有的面。請改用座標範圍 (Bounding Box) 的方式來篩選出 Z 座標最大的一端，並更新程式碼。」

### 步驟 3：執行與自我除錯迴圈

當邏輯看起來沒問題後，開始執行程式。**報錯是正常的，這正是訓練 AI (以及訓練你自己如何與 AI 溝通) 的最佳時機。**

遇到錯誤時，**不要自己動手改**，將錯誤訊息原封不動地丟回給 AI，要求它修正。

**錯誤處理 Prompt 示範：**
> 「我執行了你提供的程式碼，但在設定力量 (Force) 的步驟發生了以下錯誤：
> `AttributeError: 'Mechanical' object has no attribute 'AddForce'`
>
> 請幫我檢查這段程式碼，找出正確的 API 呼叫方式，並提供修正後可以順利執行的完整程式碼。」

### 步驟 4：持續迭代，直到完美執行

重複「執行 -> 遇到錯誤 -> 將錯誤訊息貼給 AI -> AI 提供修正程式碼 -> 再次執行」的迴圈。

直到整個腳本能夠從頭到尾順利跑完，並且輸出的物理結果（如變形量）與你過去手動操作的經驗相符為止。此時，你就獲得了一個真正屬於你、且經過實戰驗證的自動化專案範本。
